# Use cases

Esta carpeta documenta los flujos principales que el sistema debe soportar.

Cada caso de uso debe servir como guía para implementar servicios en C#/.NET.  
La idea es que el flujo quede claro antes de escribir código.

## Convención

Cada documento intenta cubrir:

- Objetivo
- Cuándo aplica
- Tablas involucradas
- Flujo esperado
- Reglas de negocio
- Cambios esperados en inventario, ventas, pagos o finanzas
- Casos especiales
- Errores esperados

## Casos incluidos

- Crear venta
- Crear pago de venta
- Crear envío de venta
- Crear reserva de producto
- Liberar reserva
- Convertir reserva en venta
- Cancelar venta
- Cancelar línea de venta
- Cambiar producto vendido
- Devolver productos después de la venta
- Aplicar descuento de campaña
- Aplicar descuento manual
- Registrar producto dañado
- Registrar producto perdido
- Registrar ajuste de inventario
- Recibir productos de una orden
- Crear movimiento financiero desde pago
- Registrar préstamo
- Registrar pago de préstamo
